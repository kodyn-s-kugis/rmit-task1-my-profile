The values that were most prevalent in the results were ambition, influence, persistence and logic; this was combined with the following motivators:

Big Five

The third test was called the Big Five; this was provided by the same provider of the previous Myers-Briggs test, truity.com. At the end of this test, the goal is to assess the following five characters (hence the name "Big Five"):

  • Openness
  • Conscientiousness
  • Extraversion
  • Agreeableness
  • Neuroticism (moods / how one is feeling and is affected by it)

Now we will review how I faired during this test, and the results for each category are outlined below:

  • Openness: 58%
  • Conscientiousness: 77%
  • Extraversion: 56%
  • Agreeableness: 62.5%
  • Neuroticism: 50%

The results corroborate the results found in the Myers-Briggs test showing major similarity in the overall assessment.

A low score on openness is indicative of concrete thinkers that avoid the unknown and the opposite being abstract. Project Idea

Overview

Although, I would usually say reinventing the wheel is not the way to go; however, demonstrating a project that was built from scratch could be used as a portfolio item to show your competency. My idea would be creating a full-fledged blog posting application that can be self-hosted by individuals. This would be an opensource project that would be posted to a Github repository. The goal isn't to be the next Facebook or even necessarily to solve an issue, but to show to employers what I can do.

Motivation

The blog posting application would be particularly interesting purely because of the challenges it's going to pose. I'll discuss later more in-depth on the requirements of the project, but the learning curve required to gather the necessary skills to accomplish this from a technical perspective would be the key interested for me. This will allow me to individually further my knowledge of web development technologies to complete a project like this.

Description / Outline

When a person goes to make a blog post, it seems like everything is seamless and straightforward, but that is far from the truth, especially with a quality approach to programming a blogging application. I'll go over the features that I believe would be key to have in my project below:

  • Opensource
  • Easy setup / deployment for the user
  • Ability to create, edit and delete blog posts
  • Inbuilt editor
  • Search functionality

Now, the critical point of the project isn't going to be making monetary value off people using the software. Some may think this is madness, but in today's age, opensource software has become highly popular within the programming community. This essentially means your software is open to be used by anyone, but you may ask why someone would do this? Well, there are lots of benefits to open-sourcing software, such as creating a community to assist you in maintaining your project or as I said earlier demonstrating to potential employers your competency. But for the user of the software, they may find this particularly important as they don't need to pay to test the software out.

The modern-day of the internet doesn't find comfort in things taking much time to complete, so the process of setting up the software must be as pain-free as possible. What this means is from when you download the software to getting it hosted with a website provider, you don't want hassles that constantly impede you before actually using it. Furthermore, since its open-source, you may have website providers creating an automated service to set this up for people.

Lots of people make mistakes or say things they may not have meant in the past, so it's important to give controls over managing a blog. The goal is to ensure the user can easily do this through three main concepts, creating a post, editing / changing a post or deleting a post.

You aren't going to have people that are going to be able to create blog posts using raw HTML and CSS as they may not know how to use the language. This is solved with an inbuilt editor so that it would work similarly to Microsoft Word, for example. The user would be capable of aligning, making text bold, embedding images and more, all from the web interface.

Finally, you have become a world-renowned blogger, and people want to search for some of your old popular posts. Like a search browser, we want to implement search functionality into the software to ensure that your follows can do this.
